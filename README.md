# 🛡️ Allievo — Parametric Income Protection for Gig Workers

> **Guidewire DEVTrails 2026 Hackathon Submission**

Allievo is a parametric income-protection insurance platform built specifically for Indian delivery partners (Swiggy, Zomato, Zepto) operating in extreme environments. 

Rather than relying on tedious manual claim paperwork, Allievo uses a **real-time disruption engine** connected to global satellite data alongside **3 bespoke Machine Learning models** to automatically detect, calculate, and instantly disburse income replacement payouts.

---

## 🏗️ The Problem Space & User Stories

**Delivery partners face extreme income volatility.** When an intense monsoon floods Mumbai, or temperature spikes above 44°C in Delhi, a delivery partner typically has to choose between risking their life for the day's wages or staying home and earning absolutely nothing. 

### User Story 1: The Delivery Partner
**Rahul**, a Zomato driver in Mumbai, pays ₹89/week for the Allievo Standard plan. 
During the afternoon, an intense monsoon cell hits his zone. The system's **OpenWeatherMap Integration** automatically registers 45mm/hr of rainfall.
Rahul doesn't need to take photos, fill out paperwork, or talk to an agent. By simply opening his app, the **Live Monitor** flags the disruption, parses his claims through an Anomaly Detection model within 4.2 seconds, and automatically credits 70% of his daily declared income to his UPI wallet so he can safely stay home!

### User Story 2: The Insurer / Admin
**Priya**, a risk manager at the insurance firm, logs into the Admin Dashboard. Because gig workers are traditionally deemed uninsurable due to massive fraud risks, Priya relies on the platform’s **Mahalanobis Fraud Engine**, which holds/flags claims displaying suspicious geospatial drops or rapid-fire claim frequencies, while letting valid claims pass instantly. Additionally, she receives real-time LLM-generated predictive actuarial summaries generated by Claude AI.

## ⚙️ Core Technology & AI Integrations

The system is powered by two distinct forms of AI running in tandem:

### 1. Hardcore Machine Learning (Deterministic)
The backend does not rely on simple `if-else` rules for finance. It uses 3 proprietary ML models written in `mlModels.js`:
- **Risk Scoring Engine (Logistic Regression):** Evaluates 7 distinct inputs (city flood risk, AQI baselines, years active, normalized income) and outputs a precise 1-100 `riskScore` by running gradient descent parameters. This controls dynamic weekly premium pricing.
- **Fraud Detection Engine (Anomaly Detection):** Calculates Mahalanobis distance from a learned "normal" centroid. Looking at duplicate density, seasonal mismatch, and GPS spoofing, it places workers in 4 automated approval tiers.
- **Payout Prediction (Linear Regression):** Mathematically deduces the safest equitable payout proportion bounded strictly by a worker's declared income.

### 2. Live Weather Oracles (OpenWeatherMap)
The system leverages the OpenWeatherMap API architecture via an internal background polling cron loop.
- **Heavy Rain** (Parsed from live `mm/hr` rainfall)
- **Extreme Heat** (Parsed from live `°C`)
- **Severe AQI** (Parsed from live `PM2.5` hazardous scale)
- *Note: In cases of Network timeouts or missing API Keys, the Node server gracefully fails back to a random simulator engine.*

### 3. Generative AI (Anthropic Claude)
- **Actuarial Prediction Insights:** Generates predictive text summaries tracking financial exposure risks.
- **Worker Motivational Feedback:** Creates unique, psychologically encouraging insights for drivers on their dashboard based on their track record.

## 🚀 Installation & Quick Start

### Prerequisites
- Node.js 18+ (Requires native `fetch` module support)
- OpenWeatherMap API Key (Free tier works perfectly)
- Anthropic Claude API Key (Optional, uses demo fallback if empty)

### 1. Configuration Setup
Clone the repository and set up your master environment variables:
```bash
git clone https://github.com/yourname/allievo.git
cd allievo

# Setup the environment file for the backend
cp .env.example backend/.env
```

Open `backend/.env` and inject your OpenWeatherMap API key:
```env
WEATHER_API_KEY=your_openweathermap_key_here
PORT=3001
DEMO_MODE=true
```

### 2. Starting the Backend Server
The backend handles the ML algorithms and the background cron disruption tracker:
```bash
cd backend
npm install
npm run dev
```

### 3. Starting the Frontend UI
The frontend handles the worker UX and the insurer analytics dashboard. Open a *new* terminal:
```bash
cd frontend
npm install
npm run dev
```

Visit the app at 👉 `http://localhost:5173`
